# 增強系統觀測力與自愈能力方案

## 目標
解決用戶提出的「觀測不夠直白」、「自愈能力不足」問題，提升系統在故障時的自我診斷與引導修復能力。

## 擬實施變更

### 1. 升級 `cmd/check.sh` 為「診斷引擎」
不再僅僅顯示 "✓/✗"，而是根據失敗組合給出**直白結論**：
- **場景 A**: 直連通，代理不通 → 結論：節點失效或訂閱解析錯誤。
- **場景 B**: 直連不通，代理通 → 結論：分流規則配置錯誤（國內流量誤入代理且代理受限）。
- **場景 C**: 兩者均不通但 TUN 接口存在 → 結論：TUN 路由或防火牆阻斷。
- **場景 D**: DNS 解析失敗 but Ping IP 成功 → 結論：DNS 規則或 Bootstrap DNS 配置錯誤。

### 2. 增加「分流洩漏」檢測
在 `check.sh` 中增加一個專門的測試：
- 檢查 `223.5.5.5`（阿里 DNS）的路由路徑。如果 traceroute 或路由表顯示其經過了代理（而不是直連），則明確標註為「路由規則配置缺陷」。

### 3. 強化日誌的可讀性 (Directness)
- 在 `singbox_health_check.sh` 中，如果連續失敗，不僅僅是 notify，還要在 `stdout` 和 `logger` 中留下具體的關鍵診斷結論。

### 4. 探索「一鍵修修復/重置」
- 在 `check` 發現關鍵異常時，主動提示用户運行：`sudo ./singbox-deploy.sh --generate && sudo systemctl restart sing-box`。

## 具體執行步驟
### 第一階段：診斷結論優化
修改 `cmd/check.sh` 的輸出邏輯，增加 `Diagnostic Summary` 部分。

### 第二階段：規則檢查邏輯
利用 `ip route get` 等指令在 `check.sh` 中實時校驗關鍵 IP 的出站路徑。
